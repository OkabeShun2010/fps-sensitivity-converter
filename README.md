# FPS感度変換ツール

複数のFPSタイトルの通常感度を、同じ `cm/180` になるように相互変換するWebアプリです。

ブラウザ上で動作するシンプルなツールとして、React + TypeScript + Vite で作成しています。

## 対応ゲーム

| ゲーム | yaw |
| --- | ---: |
| VALORANT | 0.07 |
| Apex Legends | 0.022 |
| Overwatch 2 | 0.0066 |
| Counter-Strike 2 | 0.022 |
| PUBG: BATTLEGROUNDS | 0.002222 |
| Battlefield 6 | 0.022 |
| Call of Duty: Black Ops 7 | 0.0066 |

## 対応範囲

初期版では通常感度のみ対応しています。

以下は対象外です。

- ADS感度
- スコープ感度
- スコープ倍率ごとの感度
- 視野角による体感差の補正
- ヒーロー、キャラクター、武器ごとの個別設定

## 主な機能

- 変換元ゲームの選択
- 変換先ゲームの選択
- ゲーム内感度の入力
- DPIの入力
- 変換後感度の表示
- 変換元eDPI、変換先eDPIの表示
- `cm/180` の表示
- 入力エラー表示
- ダークテーマのレスポンシブUI

## 使い方

1. 変換元ゲームを選択する。
2. 変換先ゲームを選択する。
3. 変換元ゲームで使用している通常感度を入力する。
4. マウスDPIを入力する。
5. 変換後感度、eDPI、`cm/180` を確認する。

## 計算基準

このツールでは、`cm/180` を基準に感度を変換します。

`cm/180` は、ゲーム内で視点を180度振り向かせるために必要なマウス移動距離です。

```text
cm/180 = 180 / (DPI * ゲーム内感度 * yaw) * 2.54
```

変換後感度は以下の式で求めます。

```text
変換後感度 = 変換元感度 * 変換元ゲームのyaw / 変換先ゲームのyaw
```

eDPIは以下の式で求めます。

```text
eDPI = DPI * ゲーム内感度
```

eDPIはゲームごとの `yaw` 差を含まないため、異なるゲーム間では単純比較しない前提です。
