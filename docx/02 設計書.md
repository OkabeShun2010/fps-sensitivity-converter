# FPS感度変換ツール 設計書

## 1. 設計方針

### 1.1 目的

`01 仕様書.md` の内容をもとに、React + TypeScript + Vite で実装するための画面構成、コンポーネント構成、データ構造、計算ロジック、バリデーション方針を定義する。

### 1.2 対象範囲

初期版では、VALORANT、Apex Legends、Overwatch 2 の通常感度を `cm/180` 基準で相互変換する。

以下は設計対象外とする。

- ADS感度
- スコープ感度
- 視野角補正
- ログイン機能
- データ保存
- 外部API連携

## 2. アプリ構成

### 2.1 アーキテクチャ

フロントエンドのみで完結する単一ページアプリケーションとする。

```text
ユーザー入力
  ↓
Reactコンポーネント
  ↓
入力値のバリデーション
  ↓
感度変換ロジック
  ↓
計算結果の表示
```

### 2.2 技術スタック


| 項目      | 技術         |
| ------- | ---------- |
| フロントエンド | React      |
| 言語      | TypeScript |
| ビルドツール  | Vite       |
| スタイリング  | CSS        |
| 実行環境    | ブラウザ       |


初期版では、状態管理ライブラリ、UIコンポーネントライブラリ、バックエンド、データベースは使用しない。

## 7. データ設計

### 7.1 ゲームデータ

`data/games.ts` に対応ゲームの定義を置く。

```ts
const games: Game[] = [
  { id: "valorant", name: "VALORANT", yaw: 0.07 },
  { id: "apex-legends", name: "Apex Legends", yaw: 0.022 },
  { id: "overwatch-2", name: "Overwatch 2", yaw: 0.0066 },
];
```

### 7.2 yaw係数の補足

`yaw` は、マウス入力1カウントあたりにゲーム内視点が何度回転するかを表す係数とする。

実際の回転量は、以下のようにゲーム内感度と `yaw` を掛け合わせて決まる。

```text
1カウントあたりの回転角度 = ゲーム内感度 * yaw
```

このため、同じDPI・同じゲーム内感度でも、ゲームごとの `yaw` が異なると実際の振り向き距離は変わる。

### 7.3 yaw係数の取得元

初期版では、以下の公開情報を参照して yaw 係数を採用する。

| ゲーム | yaw | 取得元 |
| --- | ---: | --- |
| VALORANT | 0.07 | [Valorant Sensitivity Yaw 0.07](https://valorantsensconverter.com/valorant-sensitivity-yaw/)、[SensLab Valorant to CS2 Sensitivity Converter](https://senslab.pro/sensitivity/valorant-to-cs2) |
| Apex Legends | 0.022 | [Apex Legends Calculator - JSCalc Blog](https://jscalc-blog.com/apex-legends-calculator/)、[Mouse Sensitivity and Settings in Apex Legends](http://vergeofapathy.com/mouse-sensitivity-apex/) |
| Overwatch 2 | 0.0066 | [Gaming Sensitivity Converter - GamerHardware](https://gamerhardware.org/gaming-sensitivity-converter/)、[Overwatch 2 Season 14 Guide - SensAi](https://www.sensai.games/guia/overwatch-2) |

これらの係数は感度変換ツールやコミュニティ解説で広く使われている値を採用している。

ゲーム側の公式ドキュメントで将来的に異なる値が確認できた場合は、`data/games.ts` の yaw 係数と本設計書を更新する。

### 7.4 ゲームID

ゲームIDは内部処理用の安定した文字列として扱う。


| ゲーム          | ID             |
| ------------ | -------------- |
| VALORANT     | `valorant`     |
| Apex Legends | `apex-legends` |
| Overwatch 2  | `overwatch-2`  |


## 8. 計算ロジック設計

### 8.1 関数一覧

`lib/sensitivity.ts` に以下の関数を定義する。


| 関数                           | 役割                |
| ---------------------------- | ----------------- |
| `calculateCmPer180`          | `cm/180` を計算する    |
| `convertSensitivity`         | 変換後感度を計算する        |
| `calculateEdpi`              | eDPIを計算する         |
| `calculateSensitivityResult` | 表示用の計算結果をまとめて生成する |


### 8.2 calculateCmPer180

```ts
function calculateCmPer180(dpi: number, sensitivity: number, yaw: number): number {
  return (180 / (dpi * sensitivity * yaw)) * 2.54;
}
```

### 8.3 convertSensitivity

```ts
function convertSensitivity(
  sensitivity: number,
  fromYaw: number,
  toYaw: number,
): number {
  return (sensitivity * fromYaw) / toYaw;
}
```

### 8.4 calculateEdpi

```ts
function calculateEdpi(dpi: number, sensitivity: number): number {
  return dpi * sensitivity;
}
```

### 8.5 calculateSensitivityResult

```ts
function calculateSensitivityResult(input: ParsedSensitivityInput): SensitivityResult {
  const convertedSensitivity = convertSensitivity(
    input.sensitivity,
    input.fromGame.yaw,
    input.toGame.yaw,
  );

  return {
    convertedSensitivity,
    fromEdpi: calculateEdpi(input.dpi, input.sensitivity),
    toEdpi: calculateEdpi(input.dpi, convertedSensitivity),
    cmPer180: calculateCmPer180(input.dpi, input.sensitivity, input.fromGame.yaw),
    description: `${input.fromGame.name} の感度 ${input.sensitivity} を ${input.toGame.name} に変換します。`,
  };
}
```

小数点丸めは計算関数内では行わず、表示時に行う。

## 9. バリデーション設計

### 9.1 方針

フォーム入力値は文字列で保持し、計算前に数値へ変換する。

以下の場合はエラーとする。

- 感度が未入力
- 感度が0以下
- 感度が数値ではない
- DPIが未入力
- DPIが0以下
- DPIが数値ではない
- 変換元ゲームが存在しない
- 変換先ゲームが存在しない

## 12. 状態管理設計

### 12.1 状態の保持場所

初期版では `App.tsx` の `useState` のみで状態を管理する。

管理対象は以下とする。

- 入力値
- バリデーションエラー
- 計算結果

### 12.2 再計算タイミング

入力値が変更されるたびに、以下を行う。

1. 入力値を検証する。
2. 検証成功時は計算結果を更新する。
3. 検証失敗時は計算結果を `null` にし、エラーを表示する。

## 13. テスト設計

### 13.1 計算ロジックの確認

最低限、以下を確認する。

- VALORANTからApex Legendsへ変換できること
- Apex LegendsからVALORANTへ変換できること
- Overwatch 2からVALORANTへ変換できること
- 同一ゲームでは入力感度と変換後感度が一致すること
- `cm/180` が仕様書の式どおりに計算されること

### 13.2 バリデーションの確認

最低限、以下を確認する。

- 感度が空の場合にエラーになること
- 感度が0以下の場合にエラーになること
- DPIが空の場合にエラーになること
- DPIが0以下の場合にエラーになること
- 数値以外の入力でエラーになること

### 13.3 UIの確認

最低限、以下を確認する。

- 入力値を変更すると結果が更新されること
- エラー時に結果が表示されないこと
- PC幅で2カラム表示になること
- スマートフォン幅で1カラム表示になること

## 15. 今後の拡張に備えた設計メモ

- ゲーム追加は `data/games.ts` の定義追加で対応できるようにする。
- ADS感度やスコープ感度は、通常感度とは別の入力種別として追加する。
- `cm/360` の併記は、`cm/180 * 2` で表示できる。
- 変換履歴やURL共有を追加する場合は、入力状態をシリアライズできる形に保つ。
