# FPS感度変換ツール 実装計画

## 1. 目的

`01 仕様書.md` と `02 設計書.md` をもとに、FPS感度変換ツールのMVPを実装するための作業手順を定義する。

この実装計画では、React + TypeScript + Vite を使用し、VALORANT、Apex Legends、Overwatch 2 の通常感度を `cm/180` 基準で相互変換できるWebアプリを作成する。

## 2. 実装方針

- 仕様書と設計書に記載されたMVP範囲のみ実装する。
- 初期版では通常感度のみ対応する。
- バックエンド、データベース、ログイン機能、外部APIは実装しない。
- 新しい依存関係は原則追加しない。
- 計算ロジックはUIから分離する。
- 小数点丸めは表示時のみ行う。
- 公式ロゴや商標素材は使用しない。

## Phase 3: 計算ロジックの実装

### 作業内容

- `lib/sensitivity.ts` を作成する。
- `cm/180` 計算関数を実装する。
- 感度変換関数を実装する。
- eDPI計算関数を実装する。
- 表示用の計算結果をまとめる関数を実装する。

### 実装対象

```text
src/lib/sensitivity.ts
```

### 実装する関数

```text
calculateCmPer180
convertSensitivity
calculateEdpi
calculateSensitivityResult
```

### 完了条件

- `cm/180 = 180 / (DPI * ゲーム内感度 * yaw) * 2.54` の式で計算できる。
- `変換後感度 = 変換元感度 * 変換元ゲームのyaw / 変換先ゲームのyaw` の式で計算できる。
- eDPIを `DPI * ゲーム内感度` で計算できる。
- 計算関数内では小数点丸めを行わない。

## Phase 4: バリデーションの実装

- ゲームIDが対応ゲーム内に存在するか検証する。

### 実装対象

```text

```

### 完了条件

- 異常入力時に計算処理へ進まない。

## 7. 実装時の注意点

- `01 仕様書.md` と `02 設計書.md` の範囲外の機能は追加しない。
- ADS感度、スコープ感度、履歴保存、URL共有は初期実装に含めない。
- 公式ロゴや商標素材は使わない。
- `cm/180` は `cm/360` に変更しない。
- 計算式や yaw 係数は仕様書に記載されたものを使用する。
- 入力値は文字列として管理し、計算前にバリデーションで数値化する。
- 表示時以外で小数点丸めを行わない。
