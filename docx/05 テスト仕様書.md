# FPS感度変換ツール テスト仕様書

## 1. 目的

FPS感度変換ツールのMVPが、`01 仕様書.md`、`02 設計書.md`、`03 実装計画.md` に沿って動作することを、ユーザーによる手動テストで確認する。

## 2. テスト対象

### 2.1 対象機能

- 変換元ゲームの選択
- 変換先ゲームの選択
- 追加ゲームの選択
- ゲーム内感度の入力
- DPIの入力
- 変換後感度の表示
- 変換元eDPIの表示
- 変換先eDPIの表示
- `cm/180` の表示
- 入力エラーの表示
- ダークテーマUIの表示
- PC幅とスマートフォン幅のレスポンシブ表示

### 2.2 対象外

- ADS感度
- スコープ感度
- 視野角補正
- 変換履歴保存
- URL共有
- ログイン機能
- 外部API連携
- 係数未確定または単一yawで扱いにくいゲームの変換

## 3. テスト環境


| 項目     | 内容                       |
| ------ | ------------------------ |
| 実行環境   | ローカル開発環境                 |
| 起動コマンド | `npm run dev`            |
| 確認URL  | `http://127.0.0.1:5173/` |
| 対象ブラウザ | Chrome、Edge などのモダンブラウザ   |
| 対象画面幅  | PC幅、スマートフォン幅             |


## 4. テスト手順

### 4.1 事前準備

1. ターミナルでプロジェクトフォルダを開く。
2. 以下のコマンドで依存関係が入っていることを確認する。

```text
npm install
```

1. 以下のコマンドで開発サーバーを起動する。

```text
npm run dev
```

1. ブラウザで以下のURLを開く。

```text
http://127.0.0.1:5173/
```

### 4.2 テスト実施方法

1. 正常系テストケースを上から順に実施する。
2. 異常系テストケースを上から順に実施する。
3. レイアウト確認を実施する。
4. 実際の結果が期待結果と一致した場合、結果欄に `OK` と記録する。
5. 期待結果と異なる場合、結果欄に `NG` と記録し、備考欄に再現手順と実際の表示を記録する。

### 4.3 テスト終了方法

開発サーバーを停止する場合は、起動中のターミナルで以下を実行する。

```text
Ctrl + C
```

確認メッセージが表示された場合は、`Y` を入力して Enter を押す。
