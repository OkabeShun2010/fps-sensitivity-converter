# FPS感度変換ツール 変更管理

## 1. 目的

FPS感度変換ツールに対する機能追加、仕様変更、不具合修正、ドキュメント修正を記録し、変更内容と影響範囲を追跡できるようにする。

変更ごとに、変更要求、影響範囲、対応内容、テスト結果、Gitコミットを紐づけて管理する。

## 2. 変更管理の流れ

1. 変更要求を記録する。
2. 影響範囲を確認する。
3. 対応方針を決める。
4. 実装またはドキュメント修正を行う。
5. 必要なテストを行う。
6. 作業履歴を更新する。
7. Gitにコミットする。
8. GitHubへpushする。

## 3. 変更種別


| 種別         | 内容                      |
| ---------- | ----------------------- |
| `feature`  | 新機能追加                   |
| `fix`      | 不具合修正                   |
| `docs`     | ドキュメント修正                |
| `refactor` | 挙動を変えないコード整理            |
| `style`    | 見た目やCSSの修正              |
| `test`     | テスト仕様、テストケース、確認作業の追加・修正 |
| `chore`    | 開発環境、設定、その他の作業          |


## 4. 変更状態


| 状態   | 内容                |
| ---- | ----------------- |
| 未着手  | まだ作業を開始していない      |
| 調査中  | 影響範囲や対応方針を確認している  |
| 対応中  | 実装または修正を行っている     |
| テスト中 | 動作確認やテストを行っている    |
| 完了   | テストまで完了し、Gitに保存済み |
| 保留   | いったん対応を止めている      |
| 中止   | 対応しないことにした        |

## 6. 変更詳細

### CHG-002: 対象ゲームを追加する

#### 変更概要

- 感度変換ツールの対象ゲームを追加する。
- 現在対応している `VALORANT`、`Apex Legends`、`Overwatch 2` に加えて、以下のタイトルを追加候補とする。
  - Counter-Strike 2
  - PUBG: BATTLEGROUNDS
  - Battlefield 6
  - Call of Duty: Black Ops 7

#### 背景・目的

- 対応ゲームを増やし、より多くのFPSプレイヤーが感度変換に利用できるようにする。
- ゲーム追加時のデータ設計、UI表示、テストケース追加の流れを実務に近い形で経験する。

#### 影響範囲

- `docx/01 仕様書.md`
- `docx/02 設計書.md`
- `docx/05 テスト仕様書.md`
- `README.md`
- `src/data/games.ts`
- `docx/04 作業履歴.md`
- `docx/03 実装計画.md`
- `docx/06 変更管理.md`

#### 対応方針

- 追加対象ゲームごとに、感度変換に使用する yaw 係数を調査する。
- 係数の取得元を記録し、仕様書または設計書に明記する。
- `src/data/games.ts` にゲーム定義を追加する。
- UIのセレクトボックスで追加ゲームを選択できるようにする。
- 必要に応じて補足説明、README、テスト仕様書を更新する。

#### 修正対象ファイル

- `docx/01 仕様書.md`
- `docx/02 設計書.md`
- `docx/05 テスト仕様書.md`
- `README.md`
- `src/data/games.ts`
- `docx/03 実装計画.md`
- `docx/04 作業履歴.md`
- `docx/06 変更管理.md`

#### テスト観点

- 追加したゲームが変換元ゲームとして選択できること。
- 追加したゲームが変換先ゲームとして選択できること。
- 追加ゲームから既存ゲームへ変換できること。
- 既存ゲームから追加ゲームへ変換できること。
- 同一ゲームを選択した場合、入力感度と変換後感度が一致すること。
- ゲーム数が増えてもUIが崩れないこと。
- `npm run build` が成功すること。

- `ReadLints` でエラーなし。
- N-010: Counter-Strike 2からVALORANTへの変換結果が期待値と一致した。
- N-011: VALORANTからPUBG: BATTLEGROUNDSへの変換結果が期待値と一致した。
- N-012: Battlefield 6からVALORANTへの変換結果が期待値と一致した。
- N-013: Call of Duty: Black Ops 7からVALORANTへの変換結果が期待値と一致した。

## 8. Git運用ルール

### 8.1 基本手順

変更作業後は、以下の流れでGitに保存する。

```text
git status
git add 対象ファイル
git commit -m "種別: 変更内容"
git push
```

### 8.2 コミットメッセージ例


| 種別         | 例                                            |
| ---------- | -------------------------------------------- |
| `feature`  | `feature: add copy result button`            |
| `fix`      | `fix: correct validation message`            |
| `docs`     | `docs: add change management document`       |
| `refactor` | `refactor: simplify sensitivity calculation` |
| `style`    | `style: adjust result card layout`           |
| `test`     | `test: add manual test cases`                |
| `chore`    | `chore: update project config`               |


## 9. 注意点

- 変更前に必ず影響範囲を確認する。
- 仕様変更がある場合は、必要に応じて `01 仕様書.md`、`02 設計書.md`、`05 テスト仕様書.md` も更新する。
- 作業結果は `04 作業履歴.md` に記録する。
- コミット前に `git status` で意図しないファイルが含まれていないか確認する。
- 公開したくない情報や秘密情報をコミットしない。
